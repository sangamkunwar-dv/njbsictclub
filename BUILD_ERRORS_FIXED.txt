Previous Error:
  "Type 'typeof import(...' does not satisfy the constraint 'RouteHandlerConfig...'"
  The types returned by 'GET(...)' are incompatible between these types.

Root Cause:
  Route handlers were being incorrectly wrapped with requireAdmin() middleware,
  which changed their return type from NextResponse to a Promise of NextResponse.
  Next.js expects route handlers to have a specific type signature.

Solution:
  Changed all route handler exports from arrow functions wrapped with
  requireAdmin to direct async functions that call the handler.

What Changed:

BEFORE (Incorrect):
  export const GET = (req: NextRequest, options) =>
    requireAdmin(async (authReq) => handler(authReq, options))

AFTER (Correct):
  export async function GET(req: NextRequest, { params }) {
    return handler(req, { params })
  }

Why This Works:
  - Next.js expects route handlers to be direct async functions
  - The function signature must match RouteHandlerConfig
  - The handler function internally checks admin permissions
  - Authentication is handled within the handler, not in the export
